# 🧩 3. HOW TO USE AI IN EACH STEP

## ✅ STEP 1: DEFINE (Clarity Stage)

👉 Ask AI:

* “What should a scalable backend for X include?”
* “Break this feature into modules”
* “What are edge cases?”

👉 Output:

* Feature list
* System breakdown
* Interview-level clarity

---

## ✅ STEP 2: DESIGN (YOUR STRENGTH ZONE)

👉 Ask AI:

* “Review my architecture”
* “What can go wrong in this design?”
* “Suggest improvements for scale”

👉 YOU DO:

* HLD (High-Level Design)
* LLD (Low-Level Design)
* DB schema
* API contracts

👉 Tools:

* Draw.io / Excalidraw
* Markdown diagrams

---

## ✅ STEP 3: IMPLEMENT (NO AI CODE RULE)

👉 You:

* Write code manually
* Use your experience

👉 AI Use:

* “Explain this concept quickly”
* “Why is this failing?”
* “Suggest debugging approach”

---

## ✅ STEP 4: REVIEW (GAME CHANGER 🔥)

👉 Paste YOUR code and ask:

* “Review this like a senior architect”
* “Find performance issues”
* “Security gaps?”

👉 This replaces:
❌ Team review
❌ Senior dependency
