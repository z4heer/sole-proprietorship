# Requirement Checklist

Use this before sending a final proposal.

## Business understanding
- Business name confirmed
- Main contact confirmed
- Nature of business understood
- Target users/customers understood

## Problem clarity
- Core problem identified
- Root cause discussed
- Current process documented
- Pain points listed clearly

## Goal definition
- Desired outcome defined
- Success criteria defined
- Priority level agreed

## Scope definition
- What is included listed clearly
- What is excluded listed clearly
- Required deliverables listed
- Dependencies identified

## Tools and systems
- Existing tools reviewed
- Access needs identified
- Data/files required listed
- Platform limitations noted

## Timeline
- Start date discussed
- Milestones identified
- Final deadline noted
- Review checkpoints planned

## Budget and approval
- Budget range known
- Pricing model selected
- Decision maker identified
- Approval path understood

## Risk checks
- Missing information flagged
- Delay risks discussed
- Change request process explained
- Confidentiality needs noted
