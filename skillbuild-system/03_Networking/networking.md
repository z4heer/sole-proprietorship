# 📘 **4. Skill: CPU Basics**

## 1. 🎯 One-Line Definition

CPU is the brain of the computer that executes instructions.

## 2. 🧭 Purpose

* Executes program logic
* Affects performance

## 3. 🧱 Core Concept

CPU = **Fetch → Decode → Execute**

**Analogy:**
Chef reading recipe → understanding → cooking

## 4. ⚙️ Internal Working

1. Fetch instruction from memory
2. Decode instruction
3. Execute operation

## 5. 🔄 System Flow

Program → RAM → CPU → Result

## 6. 🔗 Connected Systems

* RAM
* OS
* Processes

## 7. 💻 Practical Execution

* Monitor CPU usage (`top`, `task manager`)

## 8. ⚠️ Common Mistakes

* Ignoring CPU bottleneck → Fix: optimize code
* Infinite loops → Fix: logic validation

## 9. 🧪 Debugging Mindset

If system slow → check:

* CPU usage
* Running processes

## 10. 📌 Interview Explanation

“CPU processes instructions through fetch-decode-execute cycle and drives system computation.”

## 11. 🧠 Memory Hooks

* Trigger: **Think → Process → Output**
* Analogy: Chef

## 12. 🔁 Active Recall

* What does CPU do?
* Why high CPU usage?
* What is instruction cycle?

# 📘 **5. Skill: IP Addressing**

## 1. 🎯 One-Line Definition

A unique identifier assigned to devices on a network.

## 2. 🧭 Purpose

* Enables communication between devices

## 3. 🧱 Core Concept

IP = Address of device

**Analogy:**
Home address for delivering mail

## 4. ⚙️ Internal Working

1. Device gets IP
2. Request routed using IP
3. Destination reached

## 5. 🔄 System Flow

Device → Router → Internet → Target IP → Response

## 6. 🔗 Connected Systems

* DNS
* Routing
* TCP/IP

## 7. 💻 Practical Execution

* `ipconfig` / `ifconfig`
* Ping IP

## 8. ⚠️ Common Mistakes

* Confusing IP with domain → Fix: DNS role
* Ignoring subnet → Fix: understand network

## 9. 🧪 Debugging Mindset

If no connection → check:

* IP assigned?
* Network reachable?

## 10. 📌 Interview Explanation

“IP address uniquely identifies a device in a network to enable communication.”

## 11. 🧠 Memory Hooks

* Trigger: **Who am I on network?**
* Analogy: Home address

## 12. 🔁 Active Recall

* What is IP?
* Why needed?
* How assigned?

# 📘 **6. Skill: Linux Fundamentals**

## 1. 🎯 One-Line Definition

Linux is an operating system used to manage hardware and run applications.

## 2. 🧭 Purpose

* Widely used in servers and cloud

## 3. 🧱 Core Concept

OS = Interface between user and hardware

**Analogy:**
Manager controlling workers (hardware)

## 4. ⚙️ Internal Working

1. Accept commands
2. Manage processes
3. Allocate resources

## 5. 🔄 System Flow

User → Shell → Kernel → Hardware

## 6. 🔗 Connected Systems

* Hardware
* Processes
* File system

## 7. 💻 Practical Execution

* `ls`, `cd`, `pwd`
* `top`, `ps`
* File operations

## 8. ⚠️ Common Mistakes

* Wrong permissions → Fix: chmod
* Wrong path → Fix: check directory

## 9. 🧪 Debugging Mindset

If command fails → check:

* Permissions
* Path
* Logs

## 10. 📌 Interview Explanation

“Linux is a multi-user OS that manages system resources and provides command-line interface for operations.”

## 11. 🧠 Memory Hooks

* Trigger: **Command → Kernel → Action**
* Analogy: Manager

## 12. 🔁 Active Recall

* What is OS role?
* What is kernel?
* How commands executed?
