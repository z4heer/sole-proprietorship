# 🧠 1. Core Understanding (WH Framework)

### What

Angular is a **frontend framework** used to build **dynamic, scalable web applications** using components, services, and reactive programming.

---

### Why

To:

* Build structured UI
* Manage complex frontend logic
* Enable reusable components
* Handle state and API communication

---

### How

* UI built using **components + modules**
* Data handled via **data binding**
* Logic shared via **services (DI)**
* Navigation via **routing**
* API calls via **HTTP + RxJS**
* Performance optimized via **lazy loading + change detection**

---

### When

* Single Page Applications (SPA)
* Large-scale frontend systems
* Enterprise applications

---

### Where

* Browser (client-side)
* Integrated with backend APIs
* Deployed on cloud

---

### Who

* Frontend developers
* Full-stack developers

---

# 🎤 2. Explain for Fluency

### 1 Line

Angular builds dynamic UI using components and services.

---

### 30 Seconds

Angular is a frontend framework where applications are built using components. Data binding connects UI and logic, services handle business logic, and routing manages navigation. It uses RxJS for async operations and supports scalable architectures.

---

### 2 Minutes

Angular structures applications into components and modules. Data binding (one-way/two-way) keeps UI and logic in sync. Services use dependency injection to share logic. HTTP modules connect to backend APIs. RxJS handles asynchronous data streams. Forms manage user input, and state management ensures consistency. Performance is optimized using lazy loading and efficient change detection. Applications are built, tested, and deployed using Angular CLI and integrated with cloud systems.
