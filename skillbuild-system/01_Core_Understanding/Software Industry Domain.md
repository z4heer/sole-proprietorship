# 📘 **1. Skill: Software Industry Domain Understanding**

## 1. 🎯 One-Line Definition

Understanding how software products are built, delivered, and maintained in real-world business environments.

## 2. 🧭 Purpose

* Helps you see the **big picture**
* Understand how your role fits in system
* Used in: product companies, service companies, startups

## 3. 🧱 Core Concept

Software = **Problem → Solution → Delivery → Maintenance**

**Analogy:**
Like building a house → design → construct → maintain

## 4. ⚙️ Internal Working

1. Requirement gathering
2. Design (architecture)
3. Development
4. Testing
5. Deployment
6. Maintenance

## 5. 🔄 System Flow

Client Need → Requirement → Dev Team → Testing → Production → Users → Feedback

## 6. 🔗 Connected Systems

* Architecture
* DevOps
* Testing
* Production Support

## 7. 💻 Practical Execution

* Understand SDLC models (Agile, Waterfall)
* Use tools: Jira, Git, CI/CD pipelines

## 8. ⚠️ Common Mistakes

* Ignoring business context → Fix: always ask “why”
* Focusing only on code → Fix: understand full lifecycle

## 9. 🧪 Debugging Mindset

If feature fails → check:

* Requirement clarity
* Integration points
* Deployment issues

## 10. 📌 Interview Explanation

“Software industry works through SDLC where requirements are converted into deployable applications and maintained through iterative improvements.”

## 11. 🧠 Memory Hooks

* Trigger: **Idea → Build → Run → Fix**
* Analogy: House construction lifecycle

## 12. 🔁 Active Recall

* What is SDLC?
* Where does developer fit?
* Why maintenance is critical?

# 📘 **2. Skill: IT Infrastructure Basics**

## 1. 🎯 One-Line Definition

The physical and virtual components required to run software systems.

## 2. 🧭 Purpose

* Understand where your app actually runs
* Used in: servers, cloud, data centers

## 3. 🧱 Core Concept

Infra = **Compute + Storage + Network**

**Analogy:**
Like a restaurant kitchen → equipment + storage + staff movement

## 4. ⚙️ Internal Working

1. Servers provide compute
2. Storage stores data
3. Network connects components

## 5. 🔄 System Flow

User → Network → Server → Database → Response

## 6. 🔗 Connected Systems

* Cloud
* OS
* Networking
* DevOps

## 7. 💻 Practical Execution

* Deploy app on server (local/cloud)
* Monitor CPU, memory usage

## 8. ⚠️ Common Mistakes

* Ignoring infra limits → Fix: monitor resources
* Assuming infinite scaling → Fix: plan capacity

## 9. 🧪 Debugging Mindset

If app slow → check:

* CPU usage
* Memory
* Network latency

## 10. 📌 Interview Explanation

“IT infrastructure includes compute, storage, and networking components that host and run applications.”

## 11. 🧠 Memory Hooks

* Trigger: **Run = CPU + RAM + Network**
* Analogy: Kitchen operations

## 12. 🔁 Active Recall

* What are infra components?
* Where does app run?
* What causes slowdown?

# 📘 **3. Skill: Client-Server Architecture**

## 1. 🎯 One-Line Definition

A model where clients request services and servers provide responses.

## 2. 🧭 Purpose

* Foundation of web apps
* Used in all modern systems

## 3. 🧱 Core Concept

Client = Request
Server = Process + Response

**Analogy:**
Restaurant → Customer (client) orders → kitchen (server) prepares

## 4. ⚙️ Internal Working

1. Client sends request
2. Server processes logic
3. Database interaction
4. Response sent back

## 5. 🔄 System Flow

Browser → API → Backend → DB → Response → UI

## 6. 🔗 Connected Systems

* Networking
* APIs
* Database

## 7. 💻 Practical Execution

* Browser calling API (`fetch`, `curl`)
* Backend processing request

## 8. ⚠️ Common Mistakes

* Mixing frontend/backend logic → Fix: separation
* Ignoring latency → Fix: optimize APIs

## 9. 🧪 Debugging Mindset

If request fails → check:

* Network
* Server logs
* API response

## 10. 📌 Interview Explanation

“Client-server architecture separates request and processing systems where clients interact with servers over a network.”

## 11. 🧠 Memory Hooks

* Trigger: **Request → Process → Response**
* Analogy: Restaurant

## 12. 🔁 Active Recall

* Who initiates request?
* What does server do?
* Where is data stored?
