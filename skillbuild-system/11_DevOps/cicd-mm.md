# 🧠 1. Core Understanding (WH Framework)

### What

CI/CD is a process to **automatically build, test, and deploy code continuously**.

---

### Why

To:

* Deliver features faster
* Reduce manual errors
* Ensure code quality
* Enable continuous deployment

---

### How

* Code pushed → triggers pipeline
* Build + automated tests run
* Artifacts created
* Deployment executed (blue-green/canary)
* Monitoring + rollback if needed

---

### When

* Every code commit
* During release cycles
* Continuous delivery

---

### Where

* CI/CD tools (Jenkins, GitHub Actions)
* Cloud pipelines
* DevOps environments

---

### Who

* Developers
* DevOps engineers
* QA engineers

---

# 🎤 2. Explain for Fluency

### 1 Line

CI/CD automates build, test, and deployment.

---

### 30 Seconds

CI/CD pipelines automatically build code, run tests, and deploy applications. Tools like Jenkins or GitHub Actions trigger pipelines on code changes, ensuring quality and faster delivery.

---

### 2 Minutes

CI/CD starts when code is pushed to a repository, triggering a pipeline. The code is built and tested automatically, including unit tests, linting, and code coverage checks. If successful, artifacts are created and deployed to environments using strategies like blue-green or canary deployment. Containers may be built using Docker. Security checks and secrets management are applied. Logs and alerts monitor the deployment, and rollback mechanisms handle failures. This process ensures fast, reliable, and automated software delivery.

# 📖 4. Story-Based Encoding

> You enter an automated factory.

Pressing the **start button (trigger)** begins the process.

Raw materials (**code**) are processed (**build**).
Robots (**tests**) check quality.

Cleaners (**linting**) fix issues, and scores (**coverage**) are recorded.

Finished products (**artifacts**) are packed (**Docker build**).

They are delivered (**deployment**) using safe strategies (**blue-green/canary**).

Security vault (**secrets management**) protects sensitive data.

CCTV (**logs**) records activity, and alarms (**alerts**) detect failures.

If something goes wrong, products are returned (**rollback**).
