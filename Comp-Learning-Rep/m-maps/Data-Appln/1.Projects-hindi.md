# 1. Monolith Architecture

## Monolith kya hai

Pure frontend + backend ek application me.

### Iska use kisliye hota hai

* Beginner to intermediate projects
* Faster development
* Easier deployment

### Ye internally kaise kaam karta hai

Single application:

* UI
* APIs
* Business logic
* Database access

sab ek codebase me.

### Working Steps

Browser → Spring Boot App → Database → Response

### Mental Image

Single office building jahan sab departments andar hain.

### Real Engineer Usage

* Internal company tools
* ERP systems
* Admin panels

### Communication Practice

* “Ye modular monolith architecture use karta hai.”
* “Initial scalability ke liye monolith fast hota hai.”

---

# 2. Layered Architecture

## Layered Architecture kya hai

Application ko layers me divide karna.

### Layers

* Controller
* Service
* Repository
* Database

### Iska use kisliye hota hai

Clean code maintain karne.

### Ye internally kaise kaam karta hai

Controller request receive karta hai.
Service business logic handle karta hai.
Repository DB access karta hai.

### Working Steps

Frontend → Controller → Service → Repository → DB

### Mental Image

Restaurant flow:
Customer → Waiter → Chef → Kitchen Store

### Real Engineer Usage

Almost every enterprise Java application.

### Communication Practice

* “Separation of concerns maintain hota hai.”
* “Layered architecture testing easy banata hai.”
