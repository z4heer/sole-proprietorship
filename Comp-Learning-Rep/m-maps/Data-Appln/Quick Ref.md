```markdown id="rdbms-map"
# RDBMS

## What
- Relational Database Management System
- Stores structured data in tables
- Uses SQL for querying and transactions

## Why
- Ensure data consistency
- Manage business transactions reliably
- Support reporting and analytics

## How
- Data stored in rows and columns
- Relationships managed using keys
- SQL queries read/write data
- Transactions ensure ACID properties

## Core Components
- Tables
- Rows and Columns
- Primary/Foreign Keys
- SQL Engine
- Indexes
- Transactions

## Real-world Analogy
- Organized bank filing system
- Structured records with relationships

## Real-world Usage
- Banking systems
- ERP/CRM applications
- E-commerce orders
- Enterprise reporting

## Owner
- Database administrators
- Backend engineers
- Data engineering teams

## Consumer
- Applications
- APIs
- BI/analytics tools
- Enterprise users
```

```markdown id="cache-map"
# Cache

## What
- Temporary high-speed data storage
- Stores frequently accessed data

## Why
- Reduce response time
- Lower database load
- Improve scalability

## How
- Frequently used data stored in memory
- Requests served from cache first
- Expired data refreshed from source

## Core Components
- In-memory storage
- Key-value engine
- Expiration policies
- Replication layer

## Real-world Analogy
- Frequently used office desk drawer
- Faster than going to warehouse each time

## Real-world Usage
- Session management
- API response caching
- Database query acceleration
- CDN systems

## Owner
- Backend teams
- DevOps/SRE teams
- Platform engineers

## Consumer
- APIs
- Web applications
- Microservices
- Databases
```
