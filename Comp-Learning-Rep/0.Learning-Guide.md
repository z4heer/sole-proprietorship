# 2. How to Read Each Topic

# Step 1 — First Reading (Big Picture)

Goal:

```text id="y4n8ar"
Understand WHAT and WHY
```

Focus only on:

* definitions
* architecture diagrams
* real-world analogy
* process flow
* summary section

---

# Example

If reading Microservices:

Understand:

* why monolith fails
* why independent services help
* where queues fit
* where Kubernetes fits

Do NOT worry about:

* Kubernetes YAML
* Kafka configs
* production tuning

yet.

---

# Output Required

After first reading, answer:

```text id="n20nj7"
1. What problem does this solve?
2. Where is it used?
3. Why does industry use it?
```

# 6. How to Grasp Deeply

# Use the “5 Questions Method”

For every topic ask:

| Question | Example                  |
| -------- | ------------------------ |
| What?    | What is Redis?           |
| Why?     | Why not only DB?         |
| How?     | How cache works?         |
| Failure? | What if Redis crashes?   |
| Scale?   | How scale Redis cluster? |

This creates architect thinking.
