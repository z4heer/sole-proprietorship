## Skill matrix format

A skills matrix is most useful when it tracks proficiency and visible evidence, not just topic names. Templates commonly recommend defining the focus, listing essential skills, setting proficiency levels, and updating progress regularly. [ag5](https://www.ag5.com/free-skills-matrix-templates/)

Use these columns in **Skill Master**:

| Column | Meaning |
|---|---|
| Domain | Hardware, Networking, Angular, Python, Java, Git, CI/CD, Cloud, Behavioural |
| Core Skill | Example: Angular, Linux, Docker |
| Sub-skill | Example: Components, Routing, Git Merge, REST API |
| Category | Technical, Support, Behavioural |
| Priority | High, Medium, Low |
| Current Level | 0 to 4 |
| Target Level | 1 to 4 |
| Evidence | Notes, repo, video, diagram, project |
| Last Practiced | Date |
| Next Review | Date |
| Status | Not Started, Learning, Practicing, Applied, Explained, Interview Ready |

A 0-to-4 proficiency scale is common and useful: 0 for no knowledge, 1 for limited knowledge, 2 for workable ability, 3 for strong ability, and 4 for expert-level confidence. [aihr](https://www.aihr.com/blog/create-skills-matrix-competency-matrix/)

## Scientific practice loop

Since you want systematic learning, use one fixed loop for every topic. You already prefer active recall, analogy, explanation, and project-based proof.

Use this cycle:

- Read once from source material.
- Write short notes in your own words.
- Explain it using Feynman technique.
- Recall it without looking.
- Draw the flow or architecture.
- Build a tiny example.
- Build one real use case.
- Record yourself explaining it.
- Review after 1 day, 7 days, 21 days.

This combines active recall, explanation, visualization, and spaced review, which are widely used in skills development systems. [ag5](https://www.ag5.com/free-skills-matrix-templates/)
